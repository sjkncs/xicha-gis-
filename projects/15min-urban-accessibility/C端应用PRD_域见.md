# 「域见」—— 沉浸式城市空间探索平台
## C 端应用 PRD | 产品与算法方案

> 产品定位：重新定义人与城市空间的关系——从二维地图跨越到「身临其境」的沉浸式城市数字孪生体验

---

## 一、核心理念

**一句话愿景**：让每一条街道、每一栋建筑、每一个社区都有自己的「数字灵魂」。

用户的物理城市经验是碎片化的——开车经过的街角、等公交的站台、曾经住过的弄堂，这些空间记忆随着城市更新快速消散。我们的技术栈（OSM路网 + 楼栋GIS + 全景街景 + 语义分割 + 3DGS重建）恰好填补了这一空白：把真实的城市空间数字化后，以沉浸式的方式还给用户。

这不是「VR看房」或「全景漫游」的简单叠加，而是**空间智能 + 消费级硬件 + AIGC** 三者的深度融合。影石硬件采集真实影像，项目算法处理空间理解，AIGC 补全数字空间，最终在用户端呈现「真实 + 想象」的混合城市体验。

## 二、核心场景

### 场景一：「时光任意门」—— 城市空间记忆博物馆

**用户故事**：李阿姨想看看30年前自己结婚时住过的老房子还在不在、变成了什么样。张先生想找回小时候在城中村巷子里捉迷藏的街道。每一个在城市生活过的人，都有自己的「空间记忆」。

**产品功能**：
- 输入任意地址或在地地图上点击，直接进入该地点的沉浸式全景漫游
- 同一地点支持「2015年 / 2019年 / 2022年 / 现在」多时间切片切换，体验城市变迁
- 「记忆标签」系统：用户可以给任意空间位置打标签（"这是我妈以前开的小店"、"我们第一次约会的咖啡馆"），形成个人城市记忆地图
- AI 叙事引擎：基于该地点的历史影像 + POI变化 + 社区人口数据，AI 自动生成 30 秒的「这个地方曾经的样子」短视频脚本

**技术路径**：
- 全景图时间序列：利用项目已有的多年份百度/腾讯街景数据，构建同一坐标点的时序影像库
- 语义分割 + 变化检测：SegFormer/BiseNetV2 对比不同年份同一位置的道路/建筑/绿化变化
- 3DGS 动态重建：对重点城市地标（如南头古城、华侨城）做 3DGS 重建，支持视角自由旋转
- AI 叙事生成：通义千问 VL 理解影像内容，生成图文并茂的城市记忆故事

**数据资产复用**：
- `nanshan_communities_real_population.csv` → 社区历史沿革信息
- `baidu_streetview/` 多年份影像 → 时间切片对比
- `nanshan_poi_integrated_v3_wgs84.csv` → POI 变迁分析
- `segmentation_results/` → 空间要素历史对比

### 场景二：「我的 AI 城市眼」—— 影石硬件 + 实时分割增强现实

**用户故事**：小王戴着影石 AI 眼镜在深圳华强北逛街，镜片上实时浮现：「前方200米有一家评分4.8的咖啡馆，步行3分钟，途经城中村巷子有2处历史建筑」「这条街的人行道覆盖率仅62%，绕行建议走滨河大道侧」——他看到的不是冰冷的地图，而是一个「活的城市」。

**产品功能**：
- **实时环境感知**：影石眼镜 / 手机摄像头采集影像，边缘端实时运行轻量分割模型（ENet / Fast-SCNN），识别道路、行人、建筑、绿化、积水、障碍物
- **可达性预警**：结合 OSM 路网数据 + 项目楼栋数据，实时计算用户到目的地的「实际可达时间」（而非地图承诺时间），考虑城中村窄巷、楼梯、台阶等地图未标注路径
- **空间信息叠加**：AR 眼镜上叠加 POI 标注（店名、评分、开业时间）、建筑信息（楼龄、户型分布）、社区画像（平均房价、常驻人口密度）
- **步行安全指数**：基于实时分割的路面状态（积水、破损、遮挡），输出 0-100 的「步行舒适度评分」

**技术路径**：
- 边缘推理管线：将 DeepLabV3+ / BiseNetV2 量化部署至影石硬件（TensorRT 加速，INT8 量化，延迟 < 50ms）
- OSM 路网实时路由：NetworkX + OSMnx 库，基于用户实时位置做最短路径计算
- 可达性幻觉修正：项目核心算法 `velocity_illusion_model.py` 的 C 端化部署——用户输入目的地，算法返回「地图承诺时间」vs「你实际会走到的时间」对比
- AR 空间锚点：腾讯地图 AR 导航 SDK 绑定语义分割结果的空间坐标

**数据资产复用**：
- `nanshan_network_nodes.csv` (156,986 节点) → 精细化路网路由
- `nanshan_buildings_official.csv` → 建筑基础信息实时查询
- `nanshan_poi_integrated_v3_wgs84.csv` → POI 实时标注
- `section13_building_walkability.csv` → 各楼栋/街道步行可达性底图

### 场景三：「虚实融合打卡」—— 社交 + 文旅的空间新玩法

**用户故事**：小红书博主 @城市漫游家 发布了一条「深圳最小众的城中村美食地图」，不是图文攻略，而是带着空间坐标的沉浸式全景漫游包。粉丝打开 App 后，可以沿着她的路线「真实走一遍」，每个打卡点自动弹出她的语音讲解和滤镜增强的实景。

**产品功能**：
- **全景漫游路线**：用户创建带空间轨迹的「漫游路线」，支持全景影像 + 语音解说 + AR 滤镜叠加
- **3DGS 虚实混合**：博主可选择对路线中的核心地标做 3DGS 重建，粉丝浏览时可切换「纯实景 / 虚实混合 / AI 增强」三种模式
  - 纯实景：原始全景影像
  - 虚实混合：3DGS 重建后叠加历史影像层（"这里2020年是这样的"）
  - AI 增强：通义千问 VL 分析后自动添加标签、翻译招牌、文言文建筑说明
- **空间社交**：用户可以对路线中的任意位置点赞、评论、添加「隐藏彩蛋」（语音、文字、AR 虚拟道具）
- **打卡积分**：完成他人路线获得积分，积分可兑换「3DGS 重建特权」（对个人珍藏地点做精细化3D重建）

**技术路径**：
- 全景路线编辑工具：Web 端编辑器，支持导入 GPX 轨迹 + 全景影像自动对齐
- 3DGS 管线（Gaussian Splatting）：COLMAP SfM → 3DGS 优化 → 可交互 3D Gaussian Viewer（基于 Splatam / 3DGS-WebRTC）
- AIGC 空间滤镜：ControlNet / SDXL 对全景图做风格迁移（宫崎骏风、城中村赛博风、水墨风）
- WebRTC P2P 流媒体：全景影像流式传输，节省 CDN 成本

**数据资产复用**：
- 全景影像库 + manifest 元数据 → 路线基础素材
- POI 数据 → 自动路线叙事（途径店铺历史、招牌故事）
- 分割模型对全景图的分析结果 → 自动生成打卡点描述

### 场景四：「步行可达性真相」—— 城市规划公众参与工具

**用户故事**：陈阿姨住南山区某老旧小区，街道办说要「15分钟生活圈改造」，但她觉得实际走到菜市场要25分钟。通过 App，她可以直接看到：为什么是25分钟？是哪段路堵了？有没有更优路线？政府承诺的改造会不会真的缩短到15分钟？

**产品功能**：
- **个人版「15分钟城市」诊断器**：用户输入家庭住址，自动生成「我的15分钟生活圈」分析报告
  - 各类设施（菜市场、医院、学校、公园）实际可达性（而非理论可达性）
  - 「幻觉指数」可视化：哪些设施地图上看起来很近但实际很难走
  - 与同片区居民对比：你的可达性在社区内排第几百分位
- **改造模拟器**：用户可以模拟「如果这条路打通 / 这个菜市场重建 / 这片城中村改造」，可达性会如何变化
- **政策反馈通道**：分析结果一键生成「市民反馈报告」提交给规划和城管部门（参考深圳「i 深圳」政务平台接口）

**技术路径**：
- 可达性算法 C 端化：`velocity_illusion_model.py` 中的 `AI*_i` 可达性幻觉指数封装为 API 服务
- 楼栋级精细化分析：基于 `section13_building_walkability.csv` 的 4,121 个楼栋粒度分析结果，做横向对比
- 社区画像引擎：`nanshan_communities_real_population.csv` 的社区人口 / 房价 / 建成年份 → 社区类型识别 → 差异化可达性特征
- 改造影响模拟：基于 OSM 路网做「打通断头路」what-if 分析

**数据资产复用**：
- `section13_building_walkability.csv` → 楼栋级步行可达性基准数据
- `nanshan_network_nodes.csv` → 路网拓扑分析
- `nanshan_poi_integrated_v3_wgs84.csv` → 设施 POI 覆盖分析
- `nanshan_communities_real_population.csv` → 社区画像与横向对比

### 场景五：「空间胶囊」—— 个人 / 家庭数字遗产存档

**用户故事**：90岁的王爷爷想把自己生活了一辈子的南头古城拍成「空间传记」，留给孙子。他不会用复杂的拍摄设备，只需要对着老房子说话，眼镜自动采集全景影像，AI 自动整理成一部「家门口的四十年」——1985年的南头古城牌坊、1998年开的糖水铺、2015年翻新后的样子。

**产品功能**：
- **一键空间采集**：影石设备 + App 引导模式，用户沿着感兴趣路径行走，自动采集、处理、上传全景影像
- **AI 空间整理**：通义千问 VL 自动分析影像内容，生成结构化的「空间时间线」
  - 自动识别建筑变化（拆除 / 新建 / 翻新）
  - 自动识别业态变化（什么店开了、什么店关了）
  - 自动关联公开数据（该地块的规划变更记录）
- **空间日记**：用户可以对着任意位置「说话」，语音自动转化为带空间锚点的文字，附加在特定坐标上
- **家族空间树**：家庭成员的空间胶囊汇聚成「家族数字地图」，跨越三代人的城市记忆可视化呈现
- **遗产导出**：可导出为 NFT 数字藏品（腾讯区块链存证）或 PDF 纪念册

**技术路径**：
- 自动化全景采集管线：基于影石 SDK 二次开发，引导用户沿最优路径行走，自动拼接 + 去重 + 高质量筛选
- VLM 影像理解：NVIDIA API / 通义 VL 对全景图做结构化理解（建筑类型、招牌识别、业态分类）
- RAG + 知识图谱：关联 POI 历史变更数据 + 城市规划档案，生成空间叙事
- 数字遗产存证：腾讯区块链 / 蚂蚁链存证空间胶囊所有权

**数据资产复用**：
- 分割模型推理管线 → 全景图自动语义理解
- VLM walkability 模型 → 影像内容结构化
- OSM 历史数据 → 城市空间历史变迁对比

## 三、技术架构

### 3.1 整体技术架构

```
[用户层]
│
├─ 影石硬件端 (AI 眼镜 / 手机 / 全景相机)
│   └─ Edge 推理：ENet / Fast-SCNN 分割模型 (INT8 量化)
│
├─ 移动端 App (iOS / Android / Web)
│   ├─ 全景影像浏览 (WebGL 球面投影)
│   ├─ 3DGS 可交互渲染 (WebGPU / Three.js)
│   ├─ AR 空间叠加 (腾讯地图 AR SDK)
│   └─ 实时分割可视化 (Canvas 2D overlay)
│
[服务端]
│
├─ 采集管线 (全景采集 → 预处理 → 上传)
├─ 分割推理服务 (DeepLabV3+ / SegFormer / BiSeNetV2 — GPU 批量推理)
├─ 3DGS 重建服务 (COLMAP SfM → 3DGS 优化 → 网格导出)
├─ 可达性分析服务 (OSMnx 路由 + velocity_illusion_model)
├─ VLM 理解服务 (通义千问 VL API / NVIDIA API)
├─ AIGC 生成服务 (SDXL 风格迁移 + ControlNet + 语音合成)
└─ 空间数据服务 (PostGIS 地理空间数据库 + Redis 缓存)
│
[数据层]
│
├─ OSM 路网 (nanshan_network_nodes.csv — 156,986 节点)
├─ 楼栋数据 (nanshan_buildings_official.csv — 4,121 楼栋)
├─ POI 数据库 (nanshan_poi_integrated_v3 — 69,404 条)
├─ 社区画像 (nanshan_communities_real_population.csv — 395 社区)
├─ 多年份全景影像库 (baidu_streetview/ — 多年份时序数据)
└─ 分割结果库 (segmentation_results/ — 已推理结构化数据)
```

### 3.2 核心算法管线

**分割模型选型矩阵**：

| 模型 | 精度(mIoU) | 推理速度 | 适用场景 | 边缘部署 |
|------|-----------|---------|---------|---------|
| **BiSeNetV2** | 72.6 | **极快** | 实时AR分割（眼镜端） | ✅ INT8量化 <20ms |
| **ENet** | 68.0 | **极快** | 极低功耗设备 | ✅ INT8量化 <10ms |
| **Fast-SCNN** | 68.6 | **极快** | 手机实时分割 | ✅ INT8量化 <15ms |
| **DeepLabV3+** | 79.3 | 中等 | 服务端高质量推理 | GPU批量推理 |
| **SegFormer-B3** | **82.4** | 中等 | 服务端最高精度 | GPU批量推理 |
| **HRNet-OCR** | 81.2 | 较慢 | 离线精细分析 | GPU批量推理 |
| **SAM (ViT-H)** | 万物分割 | 较慢 | 特殊目标提取 | GPU按需调用 |

**3DGS 重建管线**：
```
影石全景影像序列
    ↓
COLMAP SfM 稀疏重建 (关键点匹配 + 相机位姿估计)
    ↓
COLMAP MVS 稠密重建 (点云生成)
    ↓
3DGS 参数初始化 (从点云采样高斯基元)
    ↓
3DGS 优化 (密度控制 + 颜色优化, ~30min/场景 on A100)
    ↓
导出 .ply / .splat 文件 (WebRTC 流式加载)
```

## 五、壁垒与差异化

1. **数据飞轮**：用户越多 → 全景数据越丰富 → 分割模型越准 → 3DGS资产越多 → 用户体验越好
2. **算法护城河**：项目已有的 `velocity_illusion_model`（可达性幻觉指数）是全国唯一基于真实街景数据量化「地图承诺 vs 实际可达」差异的算法
3. **低成本采集**：借助影石消费级硬件（AI眼镜 < ¥2000），用户即采集者，数据生产成本趋近于零
4. **时间序列壁垒**：项目已有多个年份的同一地点全景数据，这是任何新进入者需要3-5年才能积累的独家资产
